BASE

git config --global user.name blabla - устанавливает глобально имя , которое будет отображаться при комите (для все проектов)

git config --global user.email blabal@bla - устанавливает глобально почту, которая будет отображаться при комите

git init- инициализирует создание локального репозитория в текущей папке. Создается скрытая папка .git в которой хранится служебная информация о локальном репозитории

git status- проверяет наличие изменений в состоянии файлов проекта со времени последнего коммита. При наличии изменений выведет в консоль информацию о файлах в которых есть изменения

git addсообщает гиту, что нужно отслеживать изменения, внесенные в файлы, добавляя их в  staging area

git commit- создает слепок текущего состояния файлов, принято описывать комит кратким описание м изменений

git log - выводит изменения в репозитории в  хронологическом порядке (список всех комитов)

git checkout- переключает между ветками и выгружает их содержимое в рабочую директорию

git branch- создает новую ветку

git push -  отправляет все сохраненные в комитах изменения из локального репозитория на удаленный

git pull - забирает последние изменения с удаленного репозитория в локальный, при условии что они уже связаны

git merge- используется для слияние одну или нескольких веток в текущую

git clone  клонирует удаленный репозиторий на компьютер автоматически связывая его с локальной версией.


ADVANCED

git revert - откатывает комит, путем создания нового

git reset - используется для удаления файла из под версионного контроля

git rebase - перемещает кометы  с одной ветки на другую

git cherry-pick- используется для перенесения комитов в пределах репозитория

git stash- используется для временного сохранения всех изменений, которые незаконченны, таким образом очищая рабочую директорию


ABOUT .GITIGNORE

Если создать файл .gitignore в репозитории, Git использует его, чтобы определить, какие файлы и каталоги (или целые шаблоны) игнорировать, прежде чем совершать комит. Обычно он используется, чтобы избежать передачи временных файлов из рабочего каталога (продукты компиляции, файлы настроек проекта, и т.д.)
